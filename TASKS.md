# Project Tasks

项目：`GeoPhoto Globe`

当前目录：`/opt/geophoto-globe-dev`

## 当前定位

当前项目已不再是早期“单管理员图片地球原型”，而是：

一个正在从“地理照片 CMS + 公开地球页”演进为“常驻 globe 的地理多媒体展示平台”的项目。

任务优先级按两条线拆分：

1. 平台主线  
   先把 `GlobeShell`、账号、权限、项目、审核、访问控制、皮肤、多语言、媒体能力补齐。
2. 叙事扩展线  
   在平台能力稳定后，再做时间轴、章节、脚本化讲述。

## Next Priority

这是建议立即进入主线排期的事项。

### A. Viewer 基础架构

- 将当前 viewer 重构为常驻 `GlobeShell`
- 让项目、合集、媒体、故事切换不再卸载 globe
- 用 route + scope + overlay 模型替代“切到完全不同页面”
- 建立 `global / project / collection / media / story` 五类核心 scope
- 建立镜头转场、overlay 转场和状态回退机制

### B. 平台化基础能力

- 增加正式用户注册、登录、退出、密码重置
- 将当前单管理员密码模式升级为正式账号体系
- 增加角色权限模型，例如 `creator`、`editor`、`reviewer`、`admin`
- 增加账号与内容归属关系，支持多创作者或多组织
- 引入 `project` / `collection` 层级，不再只围绕全局照片表工作
- 增加内容可见性级别，例如公开、未列出、私有、组织内

### C. 访问控制与发布流程

- 将当前 `visible / hidden` 扩展为更完整的内容状态机
- 增加草稿、待审核、已发布、已拒绝、已归档等状态
- 增加提交审核、审核意见、重新提交流程
- 增加内容发布 / 下线操作日志
- 增加后台待审核列表和审核工作台
- 增加项目级访问级别：`public`、`password_protected`、`private`
- 增加项目访问密码设置、验证和会话级解锁状态

### D. 皮肤系统

- 设计正式的 skin token 结构，而不是继续堆零散主题字段
- 增加项目级默认皮肤绑定
- 增加 skin editor，支持实时预览 globe 和 overlay
- 支持复制现有皮肤作为新皮肤编辑
- 支持皮肤导入导出
- 规划皮肤发布、审核、安装和市场展示模型

建议首批皮肤控制范围：

- 字号与排版
- 文本颜色
- 面板颜色、边框、透明度
- 背景图
- 地球海洋 / 陆地配色
- 边境线 / 海岸线颜色
- 聚合点颜色
- 路线、高亮和媒体边框样式

### E. 多语言支持

- 增加 UI 文案国际化基础设施
- 增加项目默认语言和支持语言列表
- 增加内容多语言结构，覆盖项目、合集、媒体、故事、皮肤文案
- 增加语言切换逻辑和 fallback 逻辑
- 保证 viewer 切换语言时不刷新 globe

### F. 多媒体能力

- 在现有图片模型之外加入视频支持
- 设计视频上传、转码、封面帧和时长元数据流程
- 统一照片 / 视频在 viewer 和后台中的展示模型
- 设计视频在地点组和合集中的代表图策略

## Technical Debt

这些问题不一定先做，但需要持续留意。

- 当前 SQLite 更适合单机部署，平台化后需评估 PostgreSQL 迁移
- 当前媒体存储仍以本地磁盘为主，后续需抽象对象存储能力
- 当前仓库仍缺少系统化自动化测试
- 当前 viewer 还不是 shell + scope + overlay 模型
- 当前公开 `/storage` 的策略上线前需要更细访问控制
- 当前视频链路尚不存在，媒体模型仍偏图片导向
- 当前审核和运营留痕能力不足
- 当前主题系统仍是预设主题，不是完整皮肤系统
- 当前国际化能力尚未建立

## Not Recommended Right Now

以下事项可以保留，但不建议排在当前最前面：

- 微信小程序管理员端
- 复杂社交功能
- 多人实时协作编辑
- 过早开放完全自由的皮肤市场发布
- 仅为演示而继续堆前台动画效果

- 还没有常驻 `GlobeShell` viewer 架构
- 还不支持正式注册 / 登录用户体系
- 还不支持多创作者 / 多组织管理
- 还没有正式审核流
- 还没有项目 / 合集级内容结构
- 还没有项目级密码访问
- 还没有正式皮肤系统与皮肤编辑能力
- 还没有多语言支持
